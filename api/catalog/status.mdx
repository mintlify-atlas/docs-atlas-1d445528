## Overview

Returns comprehensive statistics about a user's sticker collection for a specific album. This endpoint shows:

- **Possessed stickers**: All stickers the user has collected
- **Missing stickers**: Stickers from the catalog the user still needs
- **Repeated stickers**: Stickers the user has multiple copies of
- **Collection statistics**: Total counts for quick progress tracking

## Endpoint

```
GET /api/albums/{albumId}/catalogo/estado
```

## Path Parameters

<ParamField path="albumId" type="long" required>
  The unique identifier of the album
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message
  
  **Example**: "Estado del álbum obtenido"
</ResponseField>

<ResponseField name="data" type="object">
  Collection status object
  
  <ResponseField name="data.laminasPoseidas" type="array">
    Array of all stickers the user currently possesses
    
    <ResponseField name="laminasPoseidas[].id" type="long">
      Unique identifier of the possessed sticker
    </ResponseField>
    
    <ResponseField name="laminasPoseidas[].nombre" type="string">
      Name of the sticker
    </ResponseField>
    
    <ResponseField name="laminasPoseidas[].imagen" type="string">
      Image URL (may be null)
    </ResponseField>
    
    <ResponseField name="laminasPoseidas[].fechaLanzamiento" type="date">
      Release date
    </ResponseField>
    
    <ResponseField name="laminasPoseidas[].tipoLamina" type="string">
      Type of sticker
    </ResponseField>
    
    <ResponseField name="laminasPoseidas[].album" type="object">
      Basic album information
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data.laminasFaltantes" type="array">
    Array of stickers from the catalog that the user doesn't have yet
    
    Structure matches `LaminaCatalogoResponseDTO` (id, nombre, imagen, fechaLanzamiento, tipoLamina, createdAt, updatedAt, active)
  </ResponseField>
  
  <ResponseField name="data.laminasRepetidas" type="object">
    Map of sticker names to their copy counts (only includes stickers with count > 1)
    
    **Example**: `{"Lionel Messi": 3, "ARG-5": 2}`
  </ResponseField>
  
  <ResponseField name="data.totalLaminas" type="integer">
    Total number of stickers the user possesses (including duplicates)
  </ResponseField>
  
  <ResponseField name="data.laminasFaltantesTotal" type="integer">
    Number of unique stickers still needed to complete the album
  </ResponseField>
  
  <ResponseField name="data.laminasRepetidasTotal" type="integer">
    Total count of all duplicate stickers
  </ResponseField>
</ResponseField>

<ResponseField name="timestamp" type="datetime">
  Server timestamp of the response
</ResponseField>

## Code Examples

<CodeGroup>

```bash cURL
curl -X GET https://api.example.com/api/albums/1/catalogo/estado \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/albums/1/catalogo/estado', {
  method: 'GET',
  headers: {
    'Accept': 'application/json'
  }
});

const { data } = await response.json();

// Calculate completion percentage
const catalogSize = data.totalLaminas + data.laminasFaltantesTotal - data.laminasRepetidasTotal;
const uniquePossessed = data.totalLaminas - data.laminasRepetidasTotal;
const completion = (uniquePossessed / catalogSize) * 100;

console.log(`Album completion: ${completion.toFixed(1)}%`);
console.log(`Missing ${data.laminasFaltantesTotal} stickers`);
console.log(`Have ${data.laminasRepetidasTotal} duplicates for trading`);
```

```python Python
import requests

response = requests.get('https://api.example.com/api/albums/1/catalogo/estado')
data = response.json()['data']

# Display status
print(f"Total stickers: {data['totalLaminas']}")
print(f"Still need: {data['laminasFaltantesTotal']}")
print(f"Duplicates: {data['laminasRepetidasTotal']}")

# Show which stickers are repeated
for nombre, count in data['laminasRepetidas'].items():
    print(f"  {nombre}: {count} copies")
```

- **Album completion**: 2 / (2 + 2) = 50%

## Use Cases

### 1. Progress Dashboard

Display collection progress to users:

```javascript
const { data } = await fetchStatus(albumId);
const catalogSize = data.totalLaminas + data.laminasFaltantesTotal - data.laminasRepetidasTotal;
const uniquePossessed = data.totalLaminas - data.laminasRepetidasTotal;
const percentage = (uniquePossessed / catalogSize) * 100;

return {
  completion: percentage,
  needed: data.laminasFaltantesTotal,
  forTrade: data.laminasRepetidasTotal
};
```

### 2. Trading System

Identify duplicates available for trading:

```python
status = get_status(album_id)
for nombre, count in status['laminasRepetidas'].items():
    if count > 1:
        available_for_trade = count - 1  # Keep one copy
        print(f"Can trade {available_for_trade} copies of {nombre}")
```

### 3. Wishlist Generation

Create a list of needed stickers:

```java
LaminasEstadoDTO estado = getEstado(albumId);
List<String> wishlist = estado.laminasFaltantes().stream()
    .map(LaminaCatalogoResponseDTO::nombre)
    .sorted()
    .toList();
```

## Implementation Details

The status calculation logic is in `LaminaService.java:104-149`:

1. **Fetch catalog**: Gets all catalog entries for the album
2. **Fetch possessed stickers**: Gets all stickers the user has added
3. **Identify missing**: Finds catalog entries not in possessed list
4. **Count duplicates**: Groups possessed stickers by name and filters for count > 1
5. **Calculate totals**: Sums up statistics

Key algorithm from `LaminaService.java:128-135`:

```java
// Count duplicates
Map<String, Integer> laminasRepetidas = laminasPoseidas.stream()
    .collect(Collectors.groupingBy(
        Lamina::getNombre,
        Collectors.summingInt(l -> 1)
    ))
    .entrySet().stream()
    .filter(e -> e.getValue() > 1)  // Only keep if count > 1
    .collect(Collectors.toMap(Map.Entry::getKey, Map.Entry::getValue));
```
