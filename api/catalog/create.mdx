## Overview

Creates a catalog of available stickers for a specific album. This endpoint defines the master list of all stickers that can be collected.

<Warning>
  **One-Time Operation**: You can only create a catalog ONCE per album. Attempting to create a second catalog will result in an error.
</Warning>

## Endpoint

```
POST /api/albums/{albumId}/catalogo
```

## Path Parameters

<ParamField path="albumId" type="long" required>
  The unique identifier of the album
</ParamField>

## Request Body

The request body must be an **array** of catalog entries. Each entry represents one sticker.

<ParamField body="[].nombre" type="string" required>
  Name or identifier of the sticker (e.g., "ARG-1", "Lionel Messi")
  
  **Validation**: Must not be blank
</ParamField>

<ParamField body="[].imagen" type="string">
  URL to the sticker image
  
  **Optional**: Can be null or empty
</ParamField>

<ParamField body="[].fechaLanzamiento" type="date" required>
  Release date of the sticker in ISO 8601 format (YYYY-MM-DD)
  
  **Validation**: Must not be null
</ParamField>

<ParamField body="[].tipoLamina" type="string" required>
  Type or category of the sticker
  
  **Examples**: "común", "especial", "dorada", "bronce"
  
  **Validation**: Must not be blank
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the operation was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable success message
  
  **Example**: "Catálogo creado exitosamente con 638 láminas"
</ResponseField>

<ResponseField name="data" type="array">
  Array of created catalog entries
  
  <ResponseField name="data[].id" type="long">
    Unique identifier of the catalog entry
  </ResponseField>
  
  <ResponseField name="data[].nombre" type="string">
    Name of the sticker
  </ResponseField>
  
  <ResponseField name="data[].imagen" type="string">
    Image URL (may be null)
  </ResponseField>
  
  <ResponseField name="data[].fechaLanzamiento" type="date">
    Release date in ISO 8601 format
  </ResponseField>
  
  <ResponseField name="data[].tipoLamina" type="string">
    Type of the sticker
  </ResponseField>
  
  <ResponseField name="data[].createdAt" type="datetime">
    Timestamp when the catalog entry was created
  </ResponseField>
  
  <ResponseField name="data[].updatedAt" type="datetime">

  <ResponseField name="data[].active" type="boolean">
    Indicates if the catalog entry is active

</ResponseField>

<ResponseField name="timestamp" type="datetime">
  Server timestamp of the response
</ResponseField>

## Implementation Details

The catalog creation logic is implemented in `LaminaService.java:42-60`:

1. **Validates album exists**: Throws `ResourceNotFoundException` if album not found
2. **Checks for existing catalog**: Counts active catalog entries for the album
3. **Enforces one-catalog rule**: Throws `RuntimeException` if catalog already exists
4. **Creates catalog entries**: Maps each DTO to a `LaminaCatalogo` entity and saves to database
5. **Returns response**: Maps saved entities back to response DTOs
